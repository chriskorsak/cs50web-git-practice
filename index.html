CS50 Web: Git Workflow

Starting a Project

  1. Create remote repository on Github
  2. Terminal: navigate to where you want to clone github directory on your local computer, then type: git clone (url of repository)

Git Push - upload latest local changes to remote repository

  1. Create project files on local computer (html, css, js, etc..)
  2. Terminal: git add (name of project file)
  3. Terminal: git commit -m "short message here detailing work you've done on files"
  4. Terminal: git status
    • This tells you what's currently happening in the repository
  5. Terminal: git push

Git Pull -download latest remote changes to local machine

  1. Terminal: git pull

Merge Conflicts

When pulling or pushing changes, it's possible to get a merge conflict if multiple people are working on the same file, or when mergining branches.

  1. Go to file with merge conflict and fix what's inside arrows.
  2. Between head and equals signs are your local changes
  3. Between equals sign and end arrows is remote version
  4. Remove all arrows, equal sign, and commit hash and save
  5. Add file git add (file name) and push git push -m "message"

      b = 3

      c = 3
      d = 4
      e = 5

Branching

Working on a feature branch instead of the master branch is a good idea.

  1. git branch tells you what branch you are on.
  2. git checkout -b (new branch name) creates and switches to a new branch
  3. git checkout ( branch name) switches to a different branch
  4. git merge (branch name) merges branches. Be sure to check which branch you are on first.

Other Git commands, tips, and shortcuts

  • git commit -am "message". This adds all files and commits changes in one command.
  • git log shows all commits and commit messages
  • git reset --hard (commit hash code) will revert back to a previous commit.
  • git reset --hard origin/master will revert back to the version on github.

Github Specifics

  1. You can "fork" someone else's repository and it creates a copy in your account
  2. You can make changes and submit a pull request to the original owner of the code to possibly merge your changes with theirs if they approve. This is a good way to collaborate on projects.
